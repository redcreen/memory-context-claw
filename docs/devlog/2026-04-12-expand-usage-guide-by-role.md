# Expand Usage Guide By Role

## Why

The first version of the detailed usage guide solved the placement problem, but it still assumed that every reader would want the same linear document.

That was too flat for real use.

Operators, maintainers, release owners, and ordinary users need different entry paths even when they are reading the same product manual.

## What Changed

- expanded the usage guide with explicit role-based entry points
- added role-specific sections for:
  - ordinary user
  - operator
  - maintainer
  - release owner
- added role-specific checklists and command sets in both English and Chinese guides

## Outcome

The usage guide is now easier to use as an operational manual instead of only a long reference page.

The document stack now separates three layers more cleanly:

- top-level README: short landing entry
- usage guide: role-based product manual
- testing docs: evidence and validation surfaces
