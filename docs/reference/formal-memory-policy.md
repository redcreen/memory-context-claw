# Formal Memory Policy

## Purpose

这份文档定义：

- 什么内容可以进入正式记忆层
- 什么内容只能停留在待确认层
- 什么内容应被归档或排除
- 后续数据治理、记忆升级、目录清理的统一标准

从现在开始，所有和记忆治理有关的动作，都优先遵守这份文档。

## Memory Layers

系统里的记忆分成 4 层：

1. `MEMORY.md`
   - 长期稳定记忆
   - 高优先级、会反复复用
   - 会被长期保留

2. `memory/YYYY-MM-DD.md`
   - 正式 daily memory
   - 已确认、但未必需要升到长期层
   - 适合保留阶段事实、近期确认信息

3. `pending-memory-candidates-YYYY-MM-DD.md`
   - 待确认候选层
   - 可以被看见、审阅、追踪
   - 但不进入正式记忆层

4. `memory_archive/`
   - 归档层
   - 保留历史材料、运行痕迹、已降级文件
   - 默认不再作为正式记忆消费层

## Admission Rules

### 1. `MEMORY.md`

只允许进入这类内容：

- 稳定身份信息
- 稳定称呼与偏好
- 长期规则
- 固定工作方式
- 持续有效的项目长期背景
- 多次确认过、未来高概率反复使用的事实

必须满足：

- 用户已明确确认，或事实已被反复验证
- 未来 1 个月以上大概率仍成立
- 属于“长期底座”，不是一次性过程信息

典型例子：

- `姓名：刘超（超哥）`
- `实际出生年份为 1983；身份证登记生日年份为 1982`
- `沟通风格：直接、实用、不废话`
- `main 工作原则`

不允许进入：

- 运行日志
- 临时调查结论
- 一次性排障过程
- 未确认事实
- 需要用户再次确认的值
- 只在当天有效的执行状态

### 2. `memory/YYYY-MM-DD.md`

允许进入这类内容：

- 当天确认的新事实
- 已确认但还不够“长期底座”的背景信息
- 阶段性项目结论
- 家庭/身份/偏好等新确认信息

必须满足：

- 内容已确认，不再处于待确认状态
- 对未来检索仍有价值
- 不只是启动回执、运行状态、简单寒暄

典型例子：

- `用户新增家庭与生日信息`
- `用户确认：实际出生年份为 1983；身份证登记生日年份为 1982`
- `女儿 / 儿子资料`

不允许进入：

- `ping`
- `在么`
- greeting
- startup 说明
- 运行日志
- 模型状态快照
- 未确认原值
- “可能有笔误、待确认”的身份信息

### 3. Pending Layer

这层专门存放：

- 待确认事实
- 有冲突的事实
- 低置信候选
- 需要人工复核的内容

允许进入：

- `review-daily-memory`
- `review-memory-md`
- guardrail 候选

不允许直接自动升级进正式层，除非：

- 用户明确确认
- 或后续被新事实覆盖并消歧

### 4. Archive Layer

适合进入这层的内容：

- startup / greeting / ping / availability 文件
- sync 日志
- 一次性运行产物
- 已经不该继续进入 recall 的历史噪音

进入 archive 后：

- 默认不再视为正式记忆层
- 保留备查价值
- 不再优先参与正式记忆消费

## Hard Rules

以下规则是强约束：

1. 未确认信息不得进入正式层
2. 运行噪音不得进入正式层
3. 待确认信息必须优先进入 pending，不得默认写入 `MEMORY.md` 或 `memory/YYYY-MM-DD.md`
4. 正式层发现历史脏值时，优先治理数据层，不优先依赖回答层补丁
5. 同一事实如果已有更清晰、更新、已确认版本，旧版本应降权、归档或标记冲突

## Conflict Handling

出现冲突时，按这个顺序处理：

1. 明确已确认的新事实
2. 长期正式记忆
3. 已确认 daily memory
4. 待确认候选
5. 历史噪音或旧 session 片段

例子：

- `你爱吃牛排` 已确认
- 旧 session 里有 `你爱吃面食`

处理原则：

- 新确认事实优先
- 旧泛化事实降权
- 必要时标记冲突，不直接删除历史

## Governance Checklist

每次治理时按这个顺序检查：

1. 这条内容属于哪一层？
2. 它是否已确认？
3. 它是否会被长期复用？
4. 它是不是运行噪音？
5. 它是否与已有正式事实冲突？
6. 它应该：
   - 升级
   - 保留
   - 挂起
   - 归档

## Operational Defaults

默认策略：

- `promote-memory-md`：进入 `MEMORY.md`
- `promote-daily-memory`：进入 `memory/YYYY-MM-DD.md`
- `review-*`：进入 pending
- 运行噪音：进入 archive

## What This Means Going Forward

从现在开始：

- 数据治理是常规任务
- 正式层准入有明确标准
- 待确认层与正式层必须长期分离
- 目录治理、回归升格、冲突治理都要以这份文档为准
