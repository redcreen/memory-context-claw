# Name Candidates

## What The Name Should Signal

The project is not a generic memory plugin.

这个项目不是一个泛泛的 memory 插件。

It is:

它更准确的定位是：

- an OpenClaw plugin
- 一个 OpenClaw 插件
- a context-engine layer
- 一个 context-engine 层
- memory-first
- 以长期记忆为基础
- focused on assembly, rerank, and retrieval quality
- 聚焦于上下文装配、重排和召回质量

## Candidate 1: Context Assembly Claw

Why it works:

为什么可行：

- direct and accurate
- 直白准确
- already aligned with the current implementation
- 和当前实现完全一致
- easy to understand inside the OpenClaw ecosystem
- 在 OpenClaw 生态里理解成本低

Weakness:

不足：

- sounds a bit functional and plain
- 略偏功能名，个性不强

## Candidate 2: Memory Context Claw

Why it works:

为什么可行：

- keeps both memory and context in the name
- 同时包含 memory 和 context 两个核心概念
- slightly smoother than “context assembly”
- 比 “context assembly” 稍微顺口一点

Weakness:

不足：

- less precise about the assembly/rerank job
- 对“装配 / 重排”这层表达不够精准

## Candidate 3: Recall Claw

Why it works:

为什么可行：

- short
- 短
- memorable
- 好记
- suitable if the project later expands beyond just context packing
- 如果未来能力扩大，会有更大伸缩空间

Weakness:

不足：

- too broad
- 太宽
- does not clearly signal rerank and assembly
- 无法直接表达重排和装配

## Candidate 4: Context Router Claw

Why it works:

为什么可行：

- sounds more like an orchestration layer
- 更像编排器
- good if the project later becomes a broader context-routing system
- 如果以后扩成更大的上下文路由系统，会比较合适

Weakness:

不足：

- “router” is less intuitive than “assembly”
- “router” 对普通用户没 “assembly” 直观

## Candidate 5: Memory Assembly Claw

Why it works:

为什么可行：

- strongly emphasizes memory-first design
- 更强调以长期记忆为核心
- clearer than abstract orchestration names
- 比抽象的 orchestration 命名更直观

Weakness:

不足：

- weakens the “context engine” feel
- 会弱化它是 context-engine 的感觉

## Candidate 6: Claw Context Engine

Why it works:

为什么可行：

- sounds product-like
- 更像产品名
- highlights plugin role clearly
- 更清楚地表达了插件角色

Weakness:

不足：

- generic
- 偏泛
- less distinctive as a GitHub repo name
- 作为 GitHub 仓库名辨识度一般

## Candidate 7: Long Memory Assembly

Why it works:

为什么可行：

- very understandable
- 非常好理解
- clearly tied to the long-memory use case
- 和长期记忆场景强绑定

Weakness:

不足：

- too descriptive
- 过于描述性
- does not feel strongly tied to OpenClaw
- 和 OpenClaw 的生态关联感弱

## Candidate 8: Recall Assembly Claw

Why it works:

为什么可行：

- combines recall and assembly
- 兼顾 recall 与 assembly
- fits the current pipeline well
- 和当前技术路线贴合

Weakness:

不足：

- sounds slightly technical
- 稍偏技术化

## My Recommendation

### Most practical choice

`memory-context-claw`

Reason:

原因：

- better public-facing name
- 更适合作为公开项目名
- still close to the actual function
- 和实际功能仍然足够接近
- good balance between precision and readability
- 在准确性和可读性之间更平衡

### Best alternative if you want a stronger public-facing name

`context-assembly-claw`

Reason:

原因：

- best alignment with the current implementation
- 和当前实现最一致
- least migration cost
- 迁移成本最低

### Best alternative if you want a broader future platform name

`context-router-claw`

Reason:

原因：

- better if the project grows into a more general orchestration layer
- 如果未来扩成通用上下文编排层，会更有空间
