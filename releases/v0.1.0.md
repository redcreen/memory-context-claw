# v0.1.0

[English](#english) | [中文](#中文)

## English

### Summary

`v0.1.0` is the first tagged release of `unified-memory-core`.

This release turns the project from an internal evolving repo into a versioned,
installable plugin with:

- stable fact/card-based context assembly
- memory-search workstream A-E completed
- governance and regression tooling in place
- user-facing docs, configuration docs, and release flow documented

### Highlights

- stable fact/card layer for:
  - identity
  - preference
  - rules
  - project/background facts
- `cardArtifact` fast path for high-value memory questions
- retrieval policy modes:
  - `fast-path-first`
  - `formal-memory-first`
  - `mixed-mode`
  - `search-first`
- formal memory governance tooling
- memory-search governance baseline and watchlist
- improved README, configuration guide, testsuite, and system architecture docs

### Important Note

This release does **not** claim to have fixed OpenClaw builtin `memory_search`
at the host level.

What it does provide is:

- plugin-side compensation
- fact-first consumption
- governance and regression protection

### Recommended Install

```bash
openclaw plugins install git+https://github.com/redcreen/unified-memory-core.git#v0.1.0
```

## 中文

### 概要

`v0.1.0` 是 `unified-memory-core` 的第一个正式 tag 版本。

这一版的意义是：项目从“持续演进中的内部仓库”，正式进入“有版本号、可安装、可发布”的状态。

这一版已经具备：

- 稳定的 fact/card 上下文组装能力
- memory-search workstream A-E 全部完成
- 治理与回归工具链到位
- 用户文档、配置文档、发布流程文档已整理完成

### 主要内容

- 稳定 fact/card 层已覆盖：
  - 身份
  - 偏好
  - 规则
  - 项目 / 背景事实
- 针对高价值记忆问题的 `cardArtifact` 快路径
- retrieval policy 四种模式：
  - `fast-path-first`
  - `formal-memory-first`
  - `mixed-mode`
  - `search-first`
- 正式记忆治理工具链
- memory-search 治理基线与 watchlist
- README、configuration、testsuite、system-architecture 文档整理完成

### 重要说明

这一版**不表示**已经从宿主层修好了 OpenClaw 内置 `memory_search`。

这一版真正提供的是：

- 插件层补强
- 事实优先消费
- 治理与回归保护

### 推荐安装方式

```bash
openclaw plugins install git+https://github.com/redcreen/unified-memory-core.git#v0.1.0
```
