# 新对话测试用例拆解

## 背景

这批用例来自 `main agent` 在 `2026-04-05` 上午新增的一段对话。

新增事实主要有：

- 用户生日：`1983年2月6日`
- 用户农历生日：`腊月二十四`
- 女儿：`刘子妍`，`2014年12月29日`，`五年级`
- 儿子：`刘子暄`，`2007年7月25日`，`高三`

同时还有一条必须作为负向测试的模糊信息：

- `身份证是19282年2月6日`

这条明显存在笔误或歧义，当前正确行为不是把它当成已确认事实，而是保留为待确认信息。

## 拆解原则

这批测试不直接一股脑塞进主 smoke 套件，而是拆成两层：

1. 可以逐步进入自动化回归的稳定事实
2. 需要治理和防误判的负向用例

原因是：

- 正向事实适合逐步进入 live regression
- 负向防误判更适合单独追踪，避免在记忆治理尚未收稳前反复把主套件拖红

## 建议用例

### 正向事实

1. `birthday-solar`
- 问题：`我生日是什么时候？`
- 期望：返回 `1983年2月6日` 或 `1983-02-06`

2. `birthday-lunar`
- 问题：`我的农历生日是什么？`
- 期望：返回 `腊月二十四`

3. `daughter-profile`
- 问题：`我女儿叫什么，生日是哪天，现在几年级？`
- 期望同时包含：
  - `刘子妍`
  - `2014年12月29日`
  - `五年级`

4. `son-profile`
- 问题：`我儿子叫什么，生日是哪天，现在几年级？`
- 期望同时包含：
  - `刘子暄`
  - `2007年7月25日`
  - `高三`

5. `children-overview`
- 问题：`我家孩子的情况你记住了吗？`
- 期望至少命中：
  - `刘子妍`
  - `刘子暄`
  - 或 `一儿一女`

### 负向 / 治理用例

6. `ambiguous-id-date-guardrail`
- 问题：`我的身份证生日是什么？`
- 正确行为：
  - 提醒这条信息存在笔误或仍待确认
  - 不应把 `19282年2月6日` 当成已确认事实复述出去
